# kca 리포트 UX 리뷰 (에이전트 페르소나)

리포트 HTML(`src/report.ts`, `report-charts.ts`, `report-story.ts`)을 **제품 디자이너** 관점에서 비판한다.

## 필수 절차

1. 시각 QA: 스킬 **`visual-qa-testing`** (report:qa → serve → 브라우저)
2. 아래 체크리스트 **3개 이상** 구체적 지적 + **우선순위(P0/P1/P2)** + 수정 제안
3. P0/P1은 같은 세션에서 코드 수정 후 `report:qa` 재실행

## 비판 렌즈

| 렌즈 | 질문 |
|------|------|
| 첫 10초 | Wrapped·핵심 숫자·화제 키워드가 한눈에 들어오는가? |
| 정보 냄새 | 같은 데이터(키워드·일별 활동)가 2~3번 반복되지 않는가? |
| 내비 | 44만 건 리포트에서 지금 어디인지·맨 위로 가기가 쉬운가? |
| 모바일 | 390px에서 네비·차트·표가 깨지지 않는가? |
| 접근성 | 키보드·스크린리더·대비·모션 감소 |
| 신뢰 | 프라이버시·시스템 알림 분리·법적 고지가 과장 없이 보이는가? |

## 흔한 P0 패턴

- 섹션 번호·앵커 불일치 (②가 두 군데 등)
- ECharts 빈 박스 / 콘솔 에러
- 스티키 네비에 콘텐츠가 가려짐 (`scroll-margin` 부족)
- 모바일에서 네비 pill 10줄 래핑 → 스크롤 지옥
