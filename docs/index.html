한 번에 읽히는 리포트.

KakaoTalk Chat Analyzer(kca)는 카카오톡 CSV 보내기를 스트리밍으로 읽고, Wrapped·ECharts 차트·Kiwi·TF-IDF 키워드(최대 120)·잔디 그리드가 담긴 한글 HTML 리포트를 만듭니다. 수십만 줄도 RAM에 올리지 않고, 메시지 원문은 HTML에 남기지 않습니다.

v0.3.3 ECharts · 키워드 120 npx kcachat@latest Node.js 22+
저장소 보내기 → 실행 전체 문서

터미널에서 (복사)

로컬
npx kcachat@latest "./KakaoTalk_Chat_보낸파일.csv" --local
업로드
npx kcachat@latest "./KakaoTalk_Chat_보낸파일.csv"
진행
npx kcachat@latest "./KakaoTalk_Chat_보낸파일.csv" --progress

kcachat@latest는 실행마다 kakaotalk-chat-analyzer@latest 본체를 받아 최신 CLI·리포트 UI를 씁니다. --local이면 BrewPage 업로드 없이 PC에만 index.html을 만듭니다. 대용량은 --progress, 단계별 ms는 --profile. 전체 옵션 → README

대용량도 Node.js만으로

카카오 보내기 CSV 전용 스트림 파서 · 단일 패스 집계 · 3MB 이상 Worker. 외부 DB 없이 설치 부담을 줄였습니다.

스트리밍파일 전체를 RAM에 올리지 않고 줄 단위 파싱
~0.4s합성 20만 메시지 벤치(환경·실데이터에 따라 다름)
--progress2.5만 건마다 stderr 진행 표시
Streaming

고속 집계

인코딩 샘플(512KB) 후 스트림 파싱. 메시지 본문은 통계에만 쓰고 즉시 버립니다. 키워드 맵 상한으로 메모리 폭주를 막습니다.

Report UX

프리미엄 한글 UI

Wrapped 카드·글래스 UI·다크/라이트·섹션 점프·지니·리듬 점수·응답 간격(초/분).

Privacy-first

부분 마스킹

기본은 이름 앞뒤만 남기고 가운데를 가려 방 안에서만 알아볼 수 있는 수준으로 공유합니다.

Keywords

Kiwi·TF-IDF

본문 어절 상위 120개·메시지 등장 횟수·워드클라우드. 시스템·오픈채팅 잡음은 필터합니다.

Embed-safe

BrewPage 친화

iframe에서 섹션 점프·하단 외부 링크가 리포트 탭을 비우지 않도록 처리했습니다. UI 수정 후에는 재업로드가 필요합니다.

Publish

임시 공유

기본 BrewPage 업로드. TempFile·Cloudflare 등 TTL 호스트는 README·kca --help 참고.

리포트에 들어가는 것

  • ⓪ Wrapped · 숫자 요약 · 인터랙티브 차트(워드클라우드, 시간대, 일별 히트맵, 키워드 막대+전체 표)
  • 연간 잔디(활동 기간만) · 오전/오후 CSS 막대 · 참여자 맵·랭킹
  • 지니(참여 쏠림) · 리듬 점수 · 하이라이트 · 시스템 알림 분리
  • 숫자는 만·억 표기 · BrewPage embed 안전 · 재업로드 시 UI 반영

원문 메시지·전체 URL·미사용 JSON 덩어리는 HTML에 넣지 않습니다.

시작하기: 카카오톡에서 CSV 보내기 → 터미널

  1. 카카오톡에서 해당 채팅방 열기 → 우측 상단 더보기(≡)
  2. 대화 보내기 → CSV 보내기로 파일 저장 (파일명은 보통 KakaoTalk_Chat_… 형태)
  3. Node.js 22 이상 설치 후, 저장한 CSV 경로를 넣어 위 명령 실행
  4. 로컬이면 생성 폴더의 index.html을 브라우저로 열기 · 업로드 모드면 터미널에 출력된 링크 확인

이 사이트는 GitHub Pages로 main 브랜치 푸시 시 자동 배포됩니다. 버그·아이디어는 Issues, 기여는 Pull request로 환영합니다. 카카오 공식 제품이 아니며, 보내기 형식이 바뀌면 파싱이 깨질 수 있으니 중요한 원본은 따로 보관하세요.
